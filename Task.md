Website development:

 

Please use the attached homepage design as the reference and build a working implementation of it.

Objective

Build a production-style implementation of the attached homepage design using:

    Frontend: TypeScript + React
    Containerization: Docker + docker-compose

You may choose the backend approach you think is most appropriate for the assignment. If the page is mostly static, you can still include a lightweight backend or mock API if needed to demonstrate full-stack thinking.

Important note

You are allowed to use AI coding tools such as Claude, Codex, Lovable, Cursor, GitHub Copilot, ChatGPT, or similar tools as part of your workflow.

We are not evaluating whether you write every line manually. We are evaluating:

    your engineering judgment
    how you structure the project
    code quality and maintainability
    how you translate a design into a working product
    how well you validate and refine AI-generated code
    your ability to make sensible tradeoffs

Requirements

Please implement the homepage with the following expectations:

Core requirements

    Recreate the attached design as closely as possible
    Use React with TypeScript
    Make the page responsive
    Ensure the codebase is clean, organized, and easy to understand
    Use Docker so the project can be run with docker-compose up

Expected deliverables

Your submission should include:

    Source code in a Git repository
    A README.md with setup and run instructions
    A docker-compose.yml that starts the app locally
    Any assumptions you made while implementing the design

Nice-to-have considerations

Where appropriate, we’d also like to see attention to:

    component structure and reusability
    accessibility basics
    loading states / error handling
    sensible styling approach
    performance considerations
    SEO fundamentals for a homepage
    maintainability and developer experience

Suggested scope

Please focus on building the homepage well rather than overengineering the solution.
If some parts of the design are ambiguous, make reasonable product and engineering decisions and document them briefly.

What we’d like you to submit

Please include:

    A link to the repository
    Brief setup instructions
    A short note describing:
        your architecture decisions
        any tradeoffs you made
        where AI tools were used and how you validated the output
